# Frontend Workspace Guidance

Read `../../AGENTS.md` first. This file adds frontend-specific guidance only.

## Fast Context

- Routing lives in `src/routes/` and uses `@tanstack/react-router`.
- Shared app-wide providers and router wiring live in `src/routes/__root.tsx`, `src/main.tsx`, and `src/router.ts`.
- `src/routeTree.gen.ts` is auto-generated by TanStack Router. Never edit manually.
- Reusable UI primitives live in `src/components/ui/` (Button, Card, Input, Logo, Spinner, Toast).
- App-specific composite components live in `src/components/` (ChatFeed, InputBar, Sidebar, GalleryPage).
- Layout shells and header live in `src/components/layout/` (Layout, Header, ModelSelector, ThinkingToggle).
- Settings-specific components live in `src/components/settings/` (ConnectorsSettings, GeneralSettings, SettingsTabs).
- Feature hooks and service-layer API calls live in `src/hooks/` and `src/services/`.
- Auth client setup lives in `src/lib/auth-client.ts`.
- Eden Treaty setup lives in `src/lib/api-client.ts`.
- Model-specific utility helpers live in `src/utils/`.
- Static assets (icons, images) live in `src/assets/`.
- Styling uses Tailwind CSS v4 via `@tailwindcss/vite` plugin, with CSS custom properties in `src/index.css` for theming. Do not introduce CSS-in-JS or styled-components.

## Frontend Invariants

- Do not introduce `react-router-dom`.
- Do not hardcode user-visible strings in components. Use `const { t } = useI18n()` and add keys in `apps/shared/src/i18n/pt-BR.ts` and `apps/shared/src/i18n/en.ts`.
- Prefer existing UI components in `src/components/ui/` before creating new primitives.
- Prefer route files that compose hooks and components rather than holding large amounts of business logic.
- Prefer Eden Treaty and TanStack Query for normal API access. Direct `fetch` is acceptable in service modules for streaming, uploads, or browser-native flows.
- Keep form, loading, and error states explicit in the UI.

## Testing Notes

- Unit and component tests live in `tests/unit/` and integration tests live in `tests/integration/`, mirroring the source tree structure.
- Use `tests/support/mocks/create-fetch-scenario.ts` only for React hook or jsdom tests that need to mock `fetch` behind Eden Treaty.
- Do not use the frontend fetch mock as a substitute for API contract testing. Route behavior belongs in API integration tests.

## Validation

- Unit and component changes: `bun run --filter @mangostudio/frontend test:unit`
- Integration changes: `bun run --filter @mangostudio/frontend test:integration`
- Coverage-sensitive work: `bun run --filter @mangostudio/frontend test:coverage`
- Auth or app-shell changes: `bun run test:browser:smoke`
